스프링(Spring) 기반 프로젝트에서 일관된 네이밍 컨벤션을 따르는 것은 코드의 가독성과 유지보수성을 높이는 데 매우 중요합니다. 
이번 글에서는 클래스명과 메서드명에 대한 네이밍 규칙을 정의하고, 이를 실무에서 어떻게 적용할 수 있는지 정리해보겠습니다.


## 1. 클래스 네이밍 규칙
클래스명은 해당 클래스의 역할을 가장 잘 나타낼 수 있도록 짓습니다. 또한, 클래스의 역할에 따라 적절한 접미사(Suffix) 를 붙여 가독성을 높입니다.

- DTO는 역할에 따라 Request, Response, Param, Condition 등으로 명확히 구분합니다.
- Entity 클래스는 보통 단순한 명사형으로 지정하여 User, Order, Product처럼 간결하게 표현합니다.
- Mapper 클래스는 UserMapper, OrderMapper와 같이 사용합니다.

| 역할 | 네이밍 규칙 |
| --- | --- |
| 컨트롤러 (Controller) | User<span style={{ color: "red" }}>Controller</span> |
| 서비스 (Service) | User<span style={{ color: "red" }}>Service</span> |
| 매퍼 (Mapper) | User<span style={{ color: "red" }}>Mapper</span> |
| DTO | 요청: User<span style={{ color: "red" }}>Request</span> <br/> 응답: User<span style={{ color: "red" }}>Response</span> <br /> 일반: User<span style={{ color: "red" }}>Param</span> <br /> 검색: User<span style={{ color: "red" }}>Condition</span> |
| 엔티티 (Entity) | User |


<br />


## 2. 메서드 네이밍 규칙
메서드명도 역할을 명확하게 나타낼 수 있도록 네이밍 패턴을 지정합니다.
각 클래스 유형별로 적절한 접미사 및 접두사를 사용하여 일관성을 유지합니다.

📌 컨트롤러 (Controller) 메서드 네이밍
컨트롤러 메서드는 보통 클라이언트 요청을 처리하는 역할을 합니다.

| 기능 | 네이밍 규칙 |
| --- | --- |
| 목록 조회 | user<span style={{ color: "red" }}>List</span>() |
| 단 건 조회 | user<span style={{ color: "red" }}>Details</span>() |
| 등록 | user<span style={{ color: "red" }}>Add</span>() |
| 수정 | user<span style={{ color: "red" }}>Modify</span>() |
| 삭제 | user<span style={{ color: "red" }}>Remove</span>() |

📌 서비스 (Service) 메서드 네이밍
서비스 메서드는 비즈니스 로직을 처리하며, 일반적으로 <span style={{ color: "red" }}>find</span>, <span style={{ color: "red" }}>add</span>, <span style={{ color: "red" }}>modify</span>, <span style={{ color: "red" }}>remove</span> 등의 패턴을 따릅니다.

| 기능 | 네이밍 규칙 |
| --- | --- |
| 목록 조회 | <span style={{ color: "red" }}>findAll</span>User() |
| 단 건 조회 | <span style={{ color: "red" }}>findOne</span>User() |
| 등록 | <span style={{ color: "red" }}>add</span>User() |
| 수정 | <span style={{ color: "red" }}>modify</span>User() |
| 삭제 | <span style={{ color: "red" }}>remove</span>User() |

📌 매퍼 (Mapper) 메서드 네이밍
Mapper는 보통 데이터베이스와의 직접적인 인터페이스를 담당하며, <span style={{ color: "red" }}>select</span>, <span style={{ color: "red" }}>insert</span>, <span style={{ color: "red" }}>update</span>, <span style={{ color: "red" }}>delete</span> 등의 패턴을 따릅니다.

| 기능 | 네이밍 규칙 |
| --- | --- |
| 조회 | <span style={{ color: "red" }}>select</span>User() |
| 등록 | <span style={{ color: "red" }}>insert</span>User() |
| 수정 | <span style={{ color: "red" }}>update</span>User() |
| 삭제 | <span style={{ color: "red" }}>delete</span>User() |


<br />


## 3. 정리 및 결론
위와 같은 네이밍 컨벤션을 적용하면 다음과 같은 이점을 얻을 수 있습니다.

✅ 일관성 유지: 프로젝트 내에서 코드 스타일이 통일되므로 개발자가 빠르게 이해하고 유지보수하기 쉬워집니다.
✅ 가독성 향상: 메서드명만 보고도 해당 기능을 쉽게 파악할 수 있습니다.
✅ 개발 생산성 향상: 팀원 간 코드 리뷰 및 협업이 더욱 원활해집니다.

이 네이밍 규칙을 프로젝트에 적용하면 코드 품질이 향상되고 유지보수 비용을 절감할 수 있습니다.
여러분의 프로젝트에서도 한 번 적용해보세요! 🚀
